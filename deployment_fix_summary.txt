# SoccerHub Deployment Fix Summary

## Issue
The application was failing to deploy in Tomcat 9 with the error:
"One or more listeners failed to start. Full details will be found in the appropriate container log file"
"Context [/SoccerHub_war_exploded] startup failed due to previous errors"

## Root Causes
1. **Jakarta EE vs Javax Servlet API**: The project was using Jakarta EE 6.0 (jakarta.servlet packages), but Tomcat 9 supports Servlet 4.0 (javax.servlet packages).
2. **Web.xml Version**: The web.xml file was using version 6.0, but Tomcat 9 supports up to version 4.0.
3. **Java Version**: The project was using Java 24, which might not be fully compatible with Tomcat 9.

## Changes Made

### 1. Updated web.xml
Changed the web.xml declaration from:
```xml
<web-app xmlns="https://jakarta.ee/xml/ns/jakartaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="https://jakarta.ee/xml/ns/jakartaee https://jakarta.ee/xml/ns/jakartaee/web-app_6_0.xsd"
         version="6.0">
```

To:
```xml
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">
```

### 2. Updated Servlet API Dependency
Changed the dependency in pom.xml from:
```xml
<dependency>
    <groupId>jakarta.servlet</groupId>
    <artifactId>jakarta.servlet-api</artifactId>
    <version>6.1.0</version>
    <scope>provided</scope>
</dependency>
```

To:
```xml
<dependency>
    <groupId>javax.servlet</groupId>
    <artifactId>javax.servlet-api</artifactId>
    <version>4.0.1</version>
    <scope>provided</scope>
</dependency>
```

### 3. Updated Java Version
Changed the Java version in pom.xml from 24 to 11:
```xml
<maven.compiler.target>11</maven.compiler.target>
<maven.compiler.source>11</maven.compiler.source>
```

### 4. Updated Servlet Imports
Changed all imports from jakarta.servlet to javax.servlet in the following files:
- DatabaseInitializer.java
- HomeServlet.java
- InizioServlet.java
- LoginServlet.java
- Registrazione.java
- RegistrazioneServlet.java

## Expected Result
The application should now deploy successfully in Tomcat 9 without any errors related to servlet compatibility.
